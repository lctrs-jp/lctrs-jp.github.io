Format of .cctrs Files

Naoki Nishida

December 10, 2012

The format of .cctrs files is an extended one of the format of .trs files.

Grammar

Red strings are new ones, that are not in .trs but newly introduced into .cctrs. Grey strings are not supposed in .cctrs and ignored when reading files.

spec ::= (   decl ) spec | e
decl ::= VAR   idlist | THEORY   listofthdecl | RULES   listofrules
    | STRATEGY   strategydecl | id   anylist
    | INTERPRETATION   interpretdecl | INCLUDE   id
anylist ::= e | id   anylist | string   anylist
    | (   anylist )   anylist | ,   anylist
idlist ::= e | id   idlist
listofthdecl ::= e | (   thdecl   )   listofthdecl
thdecl ::= id   idlist | EQUATIONS   eqlist
eqlist ::= e | equation   eqlist
equation ::= term   ==   term
listofrules ::= e | rule   listofrules | nat: rule   listofrules
rule ::= term   ->   term | term   ->   term   |   condlist
    | term   ->=   term | term   ->=   term   |   condlist
    | term   ->   term   [   formula  ]
condlist ::= cond | cond   ,   condlist
cond ::= term   ->   term | term   -><-   term
term ::= id | id   (   ) | id   (   termlist   )
termlist ::= term | term   ,   termlist
formula ::= true | false | id | id(termlist ) | (   formula   )
    | (!   formula) | (formula   &&   formula) | (formula   ||   formula)
strategydecl ::= INNERMOST | OUTERMOST | CONTEXTSENSITIVE   csstratlist
csstratlist ::= e | ( id   natlist )   csstratlist
natlist ::= e | nat   natlist
intlist ::= e | int   intlist
int ::= int | -nat
interpretdecl ::= solverdecl  prewrapper   postwrapper   funsemdecl   predsemdecl
solverdecl ::= (SMTSOLVER   id idlist) (LOGIC   id)
prewrapperdecl ::= e | (PREWRAPPER   id idlist)
postwrapperdecl ::= e | (POSTWRAPPER   id idlist)
funsemdecl ::= e | (FUNCTIONS   funsemlist)
predsemdecl ::= e | (PREDICATES   predsemlist)
funsemlist ::= e | ( id   smtlibexpr )   funsemlist
predsemlist ::= e | ( id   smtlibformula )   predsemlist
smtlibformula ::= true | false
    | (=   smtlibexpr smtlibexprlist )
    | (distinct   smtlibexpr smtlibexprlist )
    | ( predicates  smtlibexpr smtlibexpr )
    | (not   smtlibformula )
    | (or   smtlibformula   smtlibformulalist )
    | (and   smtlibformula   smtlibformulalist )
    | (implies   smtlibformula   smtlibformula )
    | (xor   smtlibformula   smtlibformulalist )
    | (iff   smtlibformula   smtlibformulalist )
smtlibformulalist ::= e | smtlibformula   smtlibformulalist
smtlibexpr ::= nat | args
    | (~   smtlibexpr )
    | (+   smtlibexpr   smtlibexpr )
    | (-   smtlibexpr   smtlibexpr )
smtlibexprlist ::= e | smtlibexpr   smtlibexprlist
predicates ::= < | <= | > | >=
args ::= $nat

id are non-empty sequences of characters except space, (, ), " and , ; and excluding special sequences ->, ==, ->=, -><-, |, true, false, !, &&, ||, not, and, or, implies, xor, iff, =, distinct, <, <=, >, >=, ~, +, -, $nat, and keywords CONTEXTSENSITIVE, EQUATIONS, INNERMOST, OUTERMOST, RULES, STRATEGY, THEORY, VAR INTERPRETATION, SMTSOLVER, LOGIC, PREWRAPPER, POSTWRAPPER, FUNCTIONS, PREDICATES, and INCLUDE.

string are sequences of any characters between double quotes.

nat are non-empty sequences of digits.

At least one VAR and one RULES sections are mandatory. If they are several, the union is taken and :

  • a symbol declared in a VAR section must not have been used in previous declarations, and is assumed to denote a variable in remaining declarations (in particular must not be applied to arguments)
  • a symbol occuring in a RULES section which has not been used before is assumed to denote a function symbol, and must be used afterwards always with the same arity.

Example

(VAR x y i z)
(INTERPRETATION
  (SMTSOLVER yices)
  (LOGIC LIA)
  (PREWRAPPER smt2y)
  (POSTWRAPPER y2smt)
  (FUNCTIONS
    (0 0)
    (s (+ $1 1))
    (p (- $1 1))
    (add (+ $1 $2))
  )
  (PREDICATES
    (leq (<= $1 $2))
  )
)
(RULES
  sum1(x) -> u(x,0,0)
  u(x,i,z) -> u(x,s(i),add(z,s(i))) [ leq(i,x) ]
  u(x,i,z) -> z [ ! leq(i,x) ]
  sum0(0) -> 0
  sum0(s(x)) -> add(sum0(x),s(x)) [ leq(0,x) ]
  sum0(p(x)) -> 0 [ leq(x,0) ]
  add(0,y) -> y
  add(s(x),y) -> s(add(x,y))
  add(p(x),y) -> p(add(x,y))
  s(p(x)) -> x
  p(s(x)) -> x
)


(VAR x y i z)
(INCLUDE intstructure.cctrs)
(RULES
  sum1(x) -> u(x,0,0)
  u(x,i,z) -> u(x,s(i),add(z,s(i))) [ leq(i,x) ]
  u(x,i,z) -> z [ ! leq(i,x) ]
  sum0(0) -> 0
  sum0(s(x)) -> add(sum0(x),s(x)) [ leq(0,x) ]
  sum0(p(x)) -> 0 [ leq(x,0) ]
  add(0,y) -> y
  add(s(x),y) -> s(add(x,y))
  add(p(x),y) -> p(add(x,y))
  s(p(x)) -> x
  p(s(x)) -> x
)
