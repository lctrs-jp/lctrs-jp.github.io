Crisys2cdcc : a proof system c-DCC for all-path reachability problems in Crisys2

RP 2024 submission version

This page provides a tool Crisys2cdcc, a proof system c-DCC for all-path reachability problems in Crisys2. Crisys2, Constrained Rewriting Induction SYStem (version 2), is a proof system based on Constrained Rewriting Induction [KN14,FKN17] for logically constrained term rewrite systems (LCTRSs) [KN13], the current version of which works interactively. c-DCC- [KN23jlamp] is a weakend but easily-implementable variant of DCC [CL18], a proof system for all-path reachability (APR) problems of LCTRSs.

Downloads

Requirement

  • The tool is implemented by OCaml (ver. 4.14.1), and to run the downloaded bytecode, ocaml (or later version) is necessary.
  • The tool needs an SMT solver that follows SMT-LIB 2 format. Please put the following file crisys2.config into the same directory as the tool: 
    (SMTSOLVER STDIN "xxxx")
(VERBOSE on)
    Please replace xxxx by a command for the SMT solver you use. For example, please write (SMTSOLVER STDIN "z3 -in") for Z3.

How to Use

Run crisys2cdcc from Command Line as follows: 

> ./crisys2cdcc example_2-25_sf.ari
or 
> ocamlrun ./crisys2cdcc example_2-25_sf.ari

The tool tries to prove a APR problem for starvation freedom, which is written the input file, constructing a proof tree.

Input Files

The tool takes an LCTRS written in the ARI format, together with an APR problem below the LCTRS definition:

( starvation-freedom [:fairness] term irreiducible-constant [:guard formula] [:var vars] )
The semantics of the above form is the same as that for constrained rewrite rules in the ARI format, and the only difference is that destinations corresponding to the right-hand sides of rewrite rules are restricted to irreducible constants. The notation ( starvation-freedom s c :guard φ ) represents the APR problem ⟨ s | φ ⟩ ⇒ c.

By adding :fairness, APR problems for starvation freedom are proved under process fairness.

Examples

[BK08, Example 2.24] for Starvation Freedom

Not Under Process Fairness

(format LCTRS :smtlib 2.6)
(theory Ints)

(sort Loc)
(sort State)

(fun state (-> Loc Loc Int Int State))
(fun noncrit1 Loc)
(fun wait1 Loc)
(fun crit1 Loc)
(fun noncrit2 Loc)
(fun wait2 Loc)
(fun crit2 Loc)
(fun success State)
(fun error State)

(rule (state noncrit1 p2 y c) (state wait1 p2 y c1)  :guard (and (= c1 1) (<= c 3))                        :var ((c1 Int)))
(rule (state wait1 p2 y c) (state crit1 p2 y1 c1)    :guard (and (> y 0) (= y1 (- y 1)) (= c1 0) (<= c 3)) :var ((y1 Int) (c1 Int)))
(rule (state crit1 p2 y c) (state noncrit1 p2 y1 c)  :guard (and (= y1 (+ y 1)) (<= c 3))                  :var ((y1 Int)))
(rule (state p1 noncrit2 y c) (state p1 wait2 y c1)  :guard (and (= c1 (ite (>= c 1) (+ c 1) c)) (<= c 3)) :var ((c1 Int)))

(rule (state p1 wait2 y c) (state p1 crit2 y1 c1)    :guard (and (> y 0) (= y1 (- y 1)) (= c1 (ite (>= c 1) (+ c 1) c)) (<= c 3)) :var ((y1 Int) (c1 Int)))
(rule (state p1 crit2 y c) (state p1 noncrit2 y1 c1) :guard (and (= y1 (+ y 1)) (= c1 (ite (>= c 1) (+ c 1) c)) (<= c 3)) :var ((y1 Int) (c1 Int)))

(rule (state p1 p2 y c) success )

(rule (state wait1 p2 y c) error    :guard (> c 3) )
(rule (state wait1 wait2 y c) error :guard (and (<= y 0) (> c 0)) )

(starvation-freedom (state noncrit1 noncrit2 y c) success :guard (and (= y 1) (= c 0)) :var ((y Int) (c Int)))
